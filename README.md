### **프로젝트 7: FreeRTOS 세마포어(Semaphore)를 이용한 자원 관리**
한정된 개수의 자원을 여러 작업이 안전하게 나눠 쓰는 방법 학습

---
## **프로젝트 개요 (Project Overview)**
이전 뮤텍스 프로젝트에서는 '오직 하나뿐인' 공유 자원을 보호하는 방법을 배움. 하지만 실제 시스템에서는 메모리 버퍼, 통신 채널, 처리 슬롯처럼 '여러 개이지만 한정된' 자원을 여러 작업이 나눠 써야 하는 경우가 많음.

이 프로젝트는 이러한 문제를 해결하기 위해, FreeRTOS의 **세마포어(Semaphore)**를 사용하여 한정된 개수의 자원을 여러 작업이 안전하게 공유하는 방법을 학습하는 것을 목표로 함.

**구현 목표:** 5개의 작업(직원)이 단 2개뿐인 '업무 처리 공간(자원)'을 사용하기 위해 경쟁하는 상황을 시뮬레이션하고, 세마포어가 어떻게 접근을 제어하는지 관찰.

---
## **개발 환경 및 핵심 개념 (Tech Stack & Core Concepts)**
* **개발 환경:** STM32CubeIDE, NUCLEO-F401RE
* **핵심 기술:**
    * **카운팅 세마포어 (Counting Semaphore):** 사용 가능한 자원의 개수를 세는 카운터 역할을 하는 동기화 도구. 여기에서는 카운터가 2인 세마포어를 사용.
    * **자원 풀링 (Resource Pooling):** 한정된 자원들을 모아두고(풀링), 필요한 작업에게 할당하고 반납받는 시스템을 구현.
    * **API:** `osSemaphoreNew()`, `osSemaphoreAcquire()`, `osSemaphoreRelease()`

---
## **주요 구현 내용**
1.  **세마포어 생성:** `main()` 함수에서 `osSemaphoreNew(2, 2, NULL)`을 호출하여, 최대 2개, 현재 사용 가능한 개수도 2개인 세마포어를 생성.
2.  **5개의 동일한 작업 생성:** `for` 루프를 사용하여 동일한 `WorkerTask` 함수를 실행하는 5개의 작업을 생성. 각 작업에는 1부터 5까지의 고유 ID를 인자로 전달.
3.  **WorkerTask 로직:**
    * `osSemaphoreAcquire()`: "업무 처리 공간"을 사용하기 위해 세마포어를 요청. 만약 빈 공간이 없다면, 다른 작업이 공간을 반납할 때까지 대기.
    * (획득 성공 시) 3초 동안 가상의 작업을 수행.
    * `osSemaphoreRelease()`: 작업이 끝나면 반드시 세마포어를 반납하여 다른 대기 중인 작업이 사용할 수 있도록 함.

---
## **실행 결과 및 분석**
프로그램 실행 결과, 시리얼 터미널에는 항상 최대 2개의 Worker 작업만이 "GOT A SLOT!" 메시지를 출력하고 작업을 시작하는 것을 확인.

5개의 작업이 동시에 실행되고 있음에도 불구하고, 세마포어 카운터가 0이 되면 다른 작업들은 "Waiting for a processing slot..." 메시지를 출력하며 대기 상태에 들어감. 작업을 마친 작업이 세마포어를 반납하면, 그제야 대기하던 작업 중 하나가 슬롯을 획득하고 작업을 시작.

---
## **배운 점 및 결론 (Key Takeaways)**
이번 프로젝트를 통해 뮤텍스와 세마포어의 명확한 차이점을 이해. 뮤텍스가 '소유권'을 기반으로 단일 자원을 보호하는 데 중점을 둔다면, 세마포어는 '사용 가능한 자원의 개수'를 관리하는 데 특화되어 있음.

이 세마포어 메커니즘은 데이터베이스 커넥션 풀, 스레드 풀, 또는 한정된 하드웨어 채널을 관리하는 등 실제 시스템에서 자원의 과부하를 막고 효율적으로 분배하는 데 필수적인 기술임.
